# Task Grid and Gantt Dialog Design

## Goal

タスク一覧をExcelに近い高密度テーブルへ変更し、長いタイトルで行が縦に伸びる問題を解消する。同時に、ガント上のタスク編集を右サイドバーからモーダルへ変更し、通常表示・全画面表示の両方で横軸を「日・1週・2週・1か月」から選べるようにする。

## Selected approach

タスク一覧には `@tanstack/react-table` を導入する。AG Gridほど大規模な置き換えをせず、既存のステータス編集、WBS階層、検索・フィルタ、3段階ソート、操作ボタンを維持しながら、列定義・列幅・リサイズ操作をheadless tableへ移す。

一覧のタイトルは常に1行で省略する。列幅が足りず実際に文字が切れた場合だけRadix Tooltipで全文を表示する。タイトル列を含む各列はドラッグで幅を変更でき、ヘッダーはスクロール中も見失いにくいsticky表示とする。既存のキーワード検索と状態・担当・マイルストーン・カテゴリ・種別・スプリントのフィルタは維持し、ソート中もWBSの親子階層は崩さない。

ガントのタスク編集は既存フォームと保存APIをそのまま使い、表示コンテナだけRadix Dialogへ置き換える。DialogはPortalで描画し、全画面ガントより高い重なり順にすることで、全画面中でも中央に表示する。Esc、背景クリック、閉じるボタンに対応し、読み取り専用時は保存操作を出さない。

横軸は既存のDay、Week、Monthに独自のTwo Weeksモードを追加する。Frappe Ganttの `view_modes` に14日ステップ、週単位スナップ、月見出しを持つモードを渡す。切替ボタンはガントカード内のツールバーへ配置するため、全画面表示に切り替えても同じ操作が残り、選択状態もReact stateで維持される。

## Components and data flow

- `TaskSpreadsheetTable` がTanStack Tableの列定義、幅、リサイズ、セル描画を担当する。
- `OverflowTooltipText` がDOMの `scrollWidth > clientWidth` を測定し、省略時だけ全文Tooltipを有効にする。
- タスクページは既存のフィルタ済み・階層ソート済み `rows` と編集コールバックを表へ渡す。
- `FrappeGantt` は利用可能な表示モード定義を受け取り、Reactの `zoom` 変更に合わせてvendorの `change_view_mode` を呼ぶ。
- ガントページの `selectedTaskId` と編集フォームは維持し、サイドバー用のバックドロップ・時刻refだけ削除する。

## Error handling and accessibility

- タスク保存失敗はモーダル内に既存のエラー文を表示する。
- 表ヘッダーは `aria-sort` を持ち、列リサイズハンドルにはラベルとダブルクリックで初期幅へ戻す操作を付ける。
- 省略Tooltipはマウスだけでなくフォーカスでも利用できる。
- モーダルはフォーカストラップ、Esc、タイトル・説明の関連付けをRadix Dialogへ委ねる。
- 2週間モードでも日付の移動・期間変更は週単位にスナップし、既存APIのYYYY-MM-DD保存形式を変えない。
