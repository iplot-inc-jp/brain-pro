# ナレッジ受信履歴 UI 設計

**日付:** 2026-07-17  
**対象:** Brain Pro frontend

## 目的

ipro-db WebhookでBrain Proへ届いたチャットと各種リソースを、プロジェクト利用者が受信後すぐ確認・検索できる場所をサイドメニューに用意する。

同時に、現在「背景・目的」配下へ混在しているナレッジ機能を独立した「ナレッジ」カテゴリへ移し、「背景・目的」は単体カテゴリに戻す。

## 情報設計

プロジェクトサイドメニューを次の順へ変更する。

```text
背景・目的
  └─ 背景・目的

ナレッジ
  ├─ チャット履歴
  ├─ リソース履歴
  ├─ ナレッジ取り込み
  ├─ ナレッジグラフ
  ├─ ナレッジ一覧編集
  ├─ RAG索引
  └─ ナレッジ設定
```

「チャット履歴」は `source=chat`、「リソース履歴」は初期Webhook契約に含まれる次のsourceを表示する。

- `document`
- `recording`
- `project_context`
- `project_memory`
- `tracker_task`

## 画面構成

両画面は共通の受信履歴ビューを使い、表示語彙とsource条件だけを変える。

### 上部

- ページ名と短い説明
- 最終取得時刻
- 更新ボタン
- 全文検索
- 期間ショートカット: すべて / 今日 / 7日 / 30日
- チャット: プラットフォーム絞り込み
- リソース: 種別絞り込み
- 添付ありのみ

### 一覧

新しい順のタイムラインとして表示する。時刻、送信元、ルームまたは種別、発言者またはタイトル、本文抜粋を一目で追えるようにする。

選択項目は強い枠線ではなく、背景色と左側の細いマーカーで示す。結果がない場合は、検索条件の解除方法とWebhook受信後にここへ現れることを案内する。

### 詳細

デスクトップでは一覧右側のインスペクタ、狭い画面では一覧下に表示する。

- チャット: 選択発言の前後10件を取得して会話として表示
- リソース: source、sourceRef、eventId、発生日時、metadata、本文全文を表示

## 見た目

Brain Pro既存の紺色を主色に、受信中の活動を示す控えめな琥珀色をアクセントにする。装飾的なグラデーションやカードの多重化は避け、運用ログとして読みやすい罫線、余白、等幅数字を使う。

PageHeader、Button、Inputなど既存UI部品を優先し、画面だけ別製品に見えないようにする。

## APIと状態

既存APIをそのまま利用する。

```text
GET /api/projects/:projectId/chat-history
GET /api/projects/:projectId/chat-history/facets
GET /api/projects/:projectId/chat-history/messages/:id/context
```

検索条件はAPIクエリへ変換する。最初の50件を取得し、`nextCursor` がある場合だけ「さらに読み込む」を表示する。追加読込時は既存行を保持して追記する。

URL query同期や高度な複数facet選択は初期実装に含めない。まず受信確認、検索、主要な絞り込み、前後文脈に集中する。

## エラー・空状態

- 初回取得失敗: 何が取得できなかったかを明示し、再試行ボタンを出す
- 追加取得失敗: 既存結果を残し、追加読込だけ再試行できる
- 結果0件: Webhook未受信か検索条件不一致かを説明する
- context取得失敗: 選択行は維持し、詳細欄だけ再試行できる

## アクセシビリティ

- 一覧行はbuttonとしてキーボード選択可能にする
- フォーカスリングを維持する
- 状態を色だけで表さず、ラベルとアイコンを併用する
- タッチ対象は44px以上を確保する
- ローディングとエラーはテキストでも伝える

## 非対象

- Webhook受信設定の作成・停止・secretローテーション
- receipt障害調査専用画面
- 受信データの編集・削除
- ナレッジグラフへの自動取り込み操作
